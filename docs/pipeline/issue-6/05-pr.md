## セキュリティ設計の要点
- **postCount≤9・レート制限(lastPostAt 10秒)は trips の update ルールに集約**（post create で trip の `get()` を増やさない＝§13.3 コスト規律）。レート制限は `request.time` 基準＋`lastPostAt==request.time` で serverTimestamp 強制（過去値書き込みによる回避を封じる）。
- **member 自己更新の封じ込め**: `onlyMyMemberChanged()`（members 変更を自分uidキーのみ）＋ `immutableOnSelfUpdate()`（memberIds/hostUserId/name/期間/status/colorsAssigned 不変）で、他人追放・host乗っ取り・memberIds改竄・他人members書き換えを拒否。
- **参加**: `isJoiningSelf()` は memberIds と members の追加を自分uidのみに限定。
- **host色配布**: `isHostAssigningColors()` を host限定・`colorsAssigned:false→true` 一発ゲート（再配布不可）で許可。member自己更新制限を迂回する裏口がないことをレビューで確認済み。
- メンバー上限 **12**（SPEC §4 準拠。リテラル強制）。
