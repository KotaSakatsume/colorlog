## 概要
ユーザーが自分の Humation アバターを**変更（カスタマイズ）**できるようにする。パーツ選択＋色変更＋ライブプレビュー＋保存。PR #24 の決定的アバターを土台に、`AvatarConfig` で部分上書きする。純JS＋SVGなので **Mock＋Expo Go で完結検証可能**（実機描画の目視のみ後述）。

Closes #25

## 変更
- `src/domain/avatar.ts`: `AvatarConfig`型（`{selections?, colors?, background?}` 全optional・`{}`は後方互換）＋ `AVATAR_SELECTION_SLOTS`/`AVATAR_COLOR_SLOTS` 定数。`buildMemberAvatarSvg(config?)` で `createAvatar` に渡す（`var(`焼き込み不変）。ピッカー用 `listPartsForSlot`/`buildPartPreviewSvg`（サムネも `bakeColorVars` で `var(` を残さない）。`@humation` import はここのみ。
- `src/components/member-avatar.tsx`: optional `config?`（後方互換）。
- `src/repositories/types.ts`: `AuthUser.avatarConfig?` ＋ `ProfileUpdate` 拡張。
- `mock-auth-service.ts` / `firebase-auth-service.ts`: `updateProfile` が avatarConfig を受理（Mock は subscribe 反映、Firebase は型整合・実保存は follow-up）。
- `src/app/profile/avatar.tsx`（新規）: ライブプレビュー＋スロット別パーツ横スクロール＋色パレット＋保存/リセット。**パフォ: 選択中スロットのみ描画＋useMemo**（全86枚同時描画しない）。slot は `SelectionSlotId`/`ColorSlotId` 型。
- `src/app/(tabs)/profile.tsx` / `members.tsx`: **自分のアバターに `config` を結線**（保存→subscribe で即反映＝ゴール6）。他メンバー行は undefined。`_layout.tsx` にルート＋プロフィールに導線。

## 品質ゲート
- `npx tsc --noEmit`: **0**
- `npx jest`: **104 pass**（既存88 + 追加16。`var(`不残・config反映＝seed既定と差分・null安全 を網羅）
- **新規依存なし**（既存 @humation を使用）。`@humation` import は avatar.ts のみ＝隔離維持。
