## 概要

共通ボタン（`src/components/ui-button.tsx`）の押下フィードバックを、瞬時の状態切り替えから Reanimated の spring / timing アニメーションに変更しました。これまでは押下時にスタイルが一気に切り替わるためカクついて見えていましたが、スケールと背景色を滑らかに補間することで、押下感をなめらかにします。

## 変更点

- `Pressable` を `Animated.createAnimatedComponent` でラップした `AnimatedPressable` 化
- shared value を 2 本導入
  - `scale`: spring で押下時の縮小アニメーション
  - `color`: timing で背景色補間の進捗を制御
- `interpolateColor` による背景色の補間
- 同色ケース（背景色が変わらない場合）は背景アニメーションを出さない no-op ガード
- `disabled` / `loading` 状態に応じてアニメーション値を復元する effect

## なぜ（背景・設計判断）

- 押下フィードバックが瞬時切り替えだったためカクついて見えていた問題を、Reanimated の spring / timing で解消する。
- 同色ケース（`secondary`、および `color` 指定の `primary`）は、押下時に背景色が変化しても見た目が変わらない／不自然になるため、背景アニメーションを出さず現状の挙動を維持する設計判断とした。
- babel 設定・テスト基盤は変更していない。既存方針を踏襲し、影響範囲を本コンポーネントに限定する。
