## 概要
メンバーのアバターを **Humation（`@humation/core`）の決定的キャラアバター**にし、**そのメンバーの配布色に染める**。写真が無くても全員に固定のかわいいアバターが付き、colorlog の「色＝アイデンティティ」と噛む。ユーザー承認のもと third-party 依存を追加。

Closes #23

実 SDK 54。**純JS（DOM非依存）＋ react-native-svg なので Expo Go でプレビュー可・node でロジック検証可**（Firebase と違い完全検証可能）。ただし Metro 実機での ESM 解決のみ未検証（下記）。

## 変更
- 依存追加: **`@humation/core` + `@humation/assets-humation-1` のみ**（core はランタイム依存ゼロ・芋づる依存なし）。
- `src/domain/avatar.ts`（新規・純関数）: `createAvatar(humation1, { seed: userId, background: 配布色hex })` → **残る `var(--hm-*, #fallback)` を実hexに焼き込み（最終SVGに `var(` を残さない）**。react-native-svg は CSS変数非対応なので必須。例外時 null。
- `src/components/member-avatar.tsx`（新規）: **写真 > Humation SVG（`SvgXml`）> 頭文字** の3段フォールバック（try/catch・onError）。
- 適用: `members.tsx`（swatch置換）・`profile.tsx`（photoURL/頭文字置換）。
- `jest.config.js`: `transformIgnorePatterns`（@humation を babel変換）＋ `moduleNameMapper`（dist直マップ・exports が import 条件のみのため）。
- `src/domain/avatar.test.ts`（新規・9件）: **`expect(svg).not.toContain('var(')`**（色が出ない罠の回帰）/ 決定的 / 背景に配布色 / 不正入力で null。

## ⚠️ Expo Go / 実機での確認項目（Metro ESM 解決が机上未検証）
jest は `moduleNameMapper` で解決したが、**Metro（実機/Expo Go）での `@humation/*` の exports 解決は未検証**。`npx expo start` で次を目視:
1. アバターが**描画される**（`Cannot find module` で落ちない）。
2. **造形色が出る**（黒/白化していない＝焼き込みが効いている）。
3. **背景がメンバーの配布色**になっている。
4. 生成失敗時に**頭文字へ縮退**しUIが壊れない。
→ もし描画されない/解決できない場合の対処: `metro.config.js` に `unstable_enablePackageExports: true` を追加（他依存への影響があるため**実機で検証してから**入れる）。
