## 変更
- **`storage.rules`**: `match /trips/{tripId}/{uid}/{postId}/{fileName}` に**5セグメント化**（main.jpg/thumb.jpg を `{fileName}` で受ける）。本人チェック・`<1.5MiB`・`image/jpeg` は維持。※ 4セグメントのままだとアップロード先 `.../{postId}/main.jpg` が default-deny になる不具合を修正。
- **`tests/rules/storage.rules.test.ts`**: 5セグメントパスの許可/拒否（本人 main/thumb・他人uid・非jpeg・1.5MiB境界・未認証・read）を網羅。
- **`src/repositories/firebase/photo-uploader.ts`**: `PhotoUploader.upload(images: ProcessedImages, target: {tripId,uid,postId})` に拡張（rules準拠パスを組むため）。passthrough スタブも追従。
- **`src/repositories/firebase/firebase-photo-uploader.ts`**（新規）: RNFirebase storage modular（`getStorage`/`ref`/`putFile(ref, uri, {contentType:'image/jpeg'})`/`getDownloadURL`）で main/thumb をアップロードし URL を返す。
- **`src/repositories/firebase/firebase-post-repository.ts`**: constructor に `ImageProcessor` 注入。`promotePhoto` を「process→2サイズ→決定的 postId `${uid}_${slotIndex}` でパス→**tx外** upload→URL確定→**既存** runTransaction」順に配線（§5-7）。tx内ロジック（postCount≤9/slot差し替え）不変。
- **`src/repositories/firebase/index.ts`**: passthrough を `FirebasePhotoUploader` に差し替え、ExpoImageProcessor を共有注入。
